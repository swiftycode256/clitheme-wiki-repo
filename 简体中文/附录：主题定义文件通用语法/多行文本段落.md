# 多行文本段落

主题定义文件中的部分定义支持多行文本段落。它允许你在一个定义中包括拥有多个行数的内容，或允许你定义带有空格的名称和项目。

> [!TIP]
> 如果你想在一行的开始位置输入结束语句（如`[/locale]`等），请在语句前加上`\`符号，如`\[/locale]`等。该语句必须放在一行中的最前面才会被识别并处理，否则前面的`\`符号将不会在最终内容中移除。

```plaintext
[default]
    Use 
    \[/default]
    To end the block
    Using [/default]
    will also work
[/default]
```

> [!IMPORTANT]
> 如果文本段落结尾没有额外的空行，内容结尾不会添加换行符。

## 关于缩进和边缘空格的处理

每一行的缩进会被减去多行文本段落内容内的共同空格大小（tab符号将会被视为8个空格）。这意味着只有行数之间的缩进差距将会被保留。如果你想要在所有的行之前都加上缩进，你可以[指定](可以设定的选项.md#如何设定选项)`leadtabs:<n>`或`leadspaces:<n>`选项；前者会添加`<n>`个缩进（Tab）符号，后者会添加`<n>`个空格。

多行文本段落中的结尾空格将会被忽略。如需添加末尾空格，请[启用](可以设定的选项.md#如何设定选项)`linebounds`选项并在一行中使用边界符号（如`|line content    |`）。

> [!NOTE]
> 在部分列表形式的多行段落中（如`[supported_apps]`、`[filter_cmds]`等）不会保留边缘空格和缩进，并且不允许指定控制缩进的选项。

### 样例

```plaintext
{entries}
    [entry] entry1
        [default]
            Text 1
            Text 2
                Text 3
                    Text 4
        [/default]
    [/entry]
    [entry] entry2
        [default]
            Text 1
            Text 2
                Text 3
        [/default] leadtabs:1
    [/entry]
    [entry] entry3
        [default]
    Text 1
    Text 2
    Text 3
        [/default]
    [/entry]
{/entries}
```

对应以下内容：

| `entry1` | `entry2` | `entry3` |
| --- | --- | --- |
| <pre>Text 1<br>Text 2<br>    Text 3<br>        Text 4</pre> | <pre>    Text 1<br>    Text 2<br>        Text 3</pre> | <pre>Text 1<br>Text 2<br>Text 3</pre>
